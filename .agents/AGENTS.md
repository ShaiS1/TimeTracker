# Tempo Project Guidelines

## End of Session Documentation Rule
At the end of every coding session or feature implementation, you MUST verify and update all supporting documentation for the project:
1. **README.md**: Ensure any new features, configuration options, or deployment adjustments are fully documented.
2. **ROADMAP.md**: Update the Status Board to mark completed features, adjust target releases, and add detailed logs under the Changelog section.
3. **Walkthrough / Tasks**: Make sure system walkthrough and task artifacts are synchronized and updated.
4. **Git Commit & Push**: Commit all documentation updates and push them to the remote repository.
